Магазин мебели "HomeStyle"

Сайт интернет-магазина мебели, разработанный на фреймворке Django. Проект включает в себя каталог товаров, систему корзины, управление пользователями и процесс оформления заказов.

🛠 Технологический стек
Backend: Python 3.11+, Django 4.2.9

Database: PostgreSQL

Frontend: HTML5, CSS3, Bootstrap 5, JavaScript

Caching: File-based Cache

Инструменты разработки: Django Debug Toolbar, django-extensions, Pillow (для работы с изображениями)

🚀 Функционал проекта
Каталог товаров: Просмотр категорий и отдельных товаров.

Пользователи: Регистрация, авторизация и личный профиль (реализовано через кастомную модель пользователя users.User).

Корзина: Добавление товаров, изменение количества и хранение данных.

Заказы: Оформление покупок с интеграцией в базу данных.

Админ-панель: Полное управление контентом через Django Admin.

📦 Установка и запуск
1. Клонирование репозитория и настройка окружения
Bash
# Создание виртуального окружения
python -m venv venv

# Активация окружения (Windows)
source venv/Scripts/activate
2. Установка зависимостей
pip install -r requirements.txt

(Основные зависимости: django, psycopg2-binary, pillow, django-debug-toolbar, djlint).

3. Настройка базы данных
Убедитесь, что у вас установлен PostgreSQL и создана база данных со следующими параметрами:

NAME: HS_Admin

USER: HS_Admin

PASSWORD: HSAdmin

4. Миграции и запуск
Bash
# Выполнение миграций
python manage.py migrate

# Создание суперпользователя (для доступа в /admin)
python manage.py createsuperuser

# Запуск сервера
python manage.py runserver

📂 Структура проекта
main/ — главная страница и общая логика.

goods/ — управление товарами и категориями.

users/ — кастомная модель пользователя и аутентификация.

carts/ — логика корзины покупок.

orders/ — система обработки заказов.

static/ & media/ — статические файлы (CSS/JS) и изображения товаров.

templates/ — общие шаблоны проекта.


Дополнение для раздела «Структура проекта»
app/ — ядро проекта, содержит основные настройки:

settings.py — конфигурация базы данных, кэширования, медиа-файлов и установленных приложений.

urls.py — главный файл маршрутизации, связывающий все модули (catalog, user, cart, orders) в единую систему.

main/ — базовый модуль приложения:

Отвечает за главную страницу и информационные страницы (например, "О нас").

Содержит логику отображения базовых шаблонов.

goods/ (Catalog) — модуль управления ассортиментом:

Логика отображения категорий и детальных карточек товаров.

Реализация поиска и фильтрации мебели.

users/ — модуль аутентификации:

Содержит кастомную модель пользователя.

Логика регистрации, авторизации и управления личными кабинетами клиентов.

carts/ — модуль корзины:

Обработка добавления товаров, изменения их количества и пересчета итоговой стоимости без перезагрузки страниц.

orders/ — модуль заказов:

Управление процессом оформления покупки, сбор данных о доставке и сохранение истории заказов в БД.

static/ — директория для статических ресурсов:

deps/ — внешние зависимости (Bootstrap, шрифты).

css/ — кастомные стили (включая my_footer_css.css).

js/ — скрипты для интерактивности.

media/ — папка для загружаемого контента:

Здесь хранятся изображения товаров, которые вы добавляете через админ-панель.

templates/ — общие HTML-шаблоны:

base.html — базовый скелет сайта.

includes/ — повторяющиеся элементы (например, cart_button.html).
